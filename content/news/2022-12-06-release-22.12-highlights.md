## Git diff 边栏

{{ asciinema(id="git-diff-gutter", width="94", height="25") }}

新的 git diff 边栏会追踪当前缓冲区与 git 索引相比的变化。边栏中的标记表示与 git 提交文件相比的添加、修改和删除。git diff 边栏默认启用。

也一定要试试新的 hunk 文本对象：`]g` 跳转到下一个更改块，`[g` 跳转到上一个。

## 下划线样式和颜色

<img src="/undercurl.png" style="max-width: 80%; display: block; margin-left: auto; margin-right: auto"/>

支持扩展下划线的终端可以渲染带样式（如波浪线或点线）以及带颜色的下划线。扩展下划线可用于改善 LSP 诊断信息的显示。这些可以通过主题中的新 `underline` 键进行配置：

```toml
"diagnostic.error" = { underline = { style = "curl", color = "red" } }
```

现有的 `underline` 修饰符现在是 `"line"` 样式的别名。

如果您想在尚未配置的主题中使用此功能，可以使用新的[主题继承]功能来仅修改您想要更改的作用域：

```toml
# ~/.config/helix/themes/my-onedark.toml
inherits = "onedark"

"diagnostic.error" = { fg = "errorfg", bg = "errorbg", underline = { color = "red" } }
```

[主题继承]: https://docs.helix-editor.com/master/themes.html#inheritance

## 终端失去焦点时自动保存

<img src="/auto-save.gif" style="max-width: 80%; display: block; margin-left: auto; margin-right: auto"/>

支持焦点事件的终端现在可以在您切换到新窗口时自动保存当前文件。通过将 `editor.auto-save` 配置键设置为 `true` 来启用此行为。

## 多光标补全

{{ asciinema(id="multi-cursor-completion", width="94", height="25") }}

使用多重选择编辑是 Helix 的核心功能。LSP 自动补全现在会在使用多重选择编辑时应用于所有光标，让您借助自动补全轻松替换多个符号实例。

## 缓冲区栏

{{ asciinema(id="bufferline", width="94", height="25") }}

缓冲区栏是显示在窗口顶部的缓冲区列表。它让您对正在处理的缓冲区一目了然。将 `editor.bufferline` 配置键设置为 `"always"` 以始终显示缓冲区栏，设置为 `"multiple"` 仅在打开多个缓冲区时显示，设置为 `"never"` 则禁用。

## 幕后改进

22.12 还为 Helix 的内部机制带来了重要变化：

* 文件写入的代码路径已被全面改造。这修复了关于写入大文件和写入失败的一些边界情况。
* tree-sitter 解析、查询和注入处理的性能得到了提升。这特别对于大型 Markdown 文件有明显的速度提升，因为 Markdown 广泛使用了注入。
* LSP 代码路径中的错误处理得到了改进。Helix 现在能够优雅地处理诸如语言服务器不支持某些能力以及语言服务器意外崩溃等情况。
* `:reload` 命令的速度和内存使用得到了极大改善。这得益于 [`imara-diff`] 的创建，这是一个比 `git` 内部使用的差异比较实现更快的全新实现。
