Helix 是一个模式文本编辑器，内置支持多重选择、语言服务器协议（LSP）、tree-sitter，以及实验性的调试适配器协议（DAP）支持。

今天我们迎来了 23.03 版本的发布。首先，**非常感谢**所有参与的人！这个版本包含了 102 位贡献者的更改。今天的发布带来了一些特别有用和令人兴奋的功能，所以让我们直接进入正题。

## 软换行

{{ asciinema(id="softwrap", width="94", height="25") }}

软换行将长行拆分为适合视图的多行，而不会在文本中插入实际的换行符（即所谓的"硬换行"）。这对于阅读和编辑文档或特别长的代码行非常有用。在配置中使用 `editor.soft-wrap.enable` 键启用软换行：

```toml
# ~/.config/helix/config.toml
[editor.soft-wrap]
enable = true
```

换行位置的启发式规则和换行指示器也可以进行[配置](https://docs.helix-editor.com/configuration.html#editorsoft-wrap-section)。

软换行是在 Helix 代码库中新的"虚拟文本"API 之上实现的第一个功能。虚拟文本 API 允许我们装饰和操作文档中的实际文本，而不会影响定位和修改。虚拟文本 API 为许多令人兴奋的新功能打开了大门，比如我们接下来会看到的内联提示。

## 内联提示

{{ asciinema(id="inlay-hints", width="94", height="25") }}

内联提示是最新 LSP 规范中的一项新功能。语言服务器可以为编辑器提供在文档中显示的提示。在上面的示例中，rust-analyzer 为变量和中间表达式提供了类型提示，以及函数调用中的参数名称。

内联提示可以通过配置中的 `lsp.display-inlay-hints` 键启用。

## 对代码片段的初始支持

{{ asciinema(id="snippets", width="94", height="25") }}

LSP 代码片段是语言服务器为编辑器提供更复杂补全的一种方式。语言服务器可以指定补全中光标的位置（制表位）、占位符值等。Helix 现在对 LSP 补全有了初步支持，更高级的代码片段功能如虚拟占位符、多制表位和变量替换则留待未来完成。

## 动态工作区符号选择器

{{ asciinema(id="dynamic-workspace-symbol-picker", width="94", height="25") }}

工作区符号选择器（`<space>S`）是一个由 LSP 驱动的功能，用于在项目中查找符号。然而，大型项目可能有很多符号，因此这个符号选择器通常不会一次性加载所有符号。不过，在这个版本中，工作区符号选择器已被重新设计，可以在选择器中输入时动态地使用新查询重新请求符号。这让语言服务器能够缩小符号范围，只告诉 Helix 与搜索相关的符号。这次重写修复了与某些语言服务器的兼容性问题，这些服务器在没有查询时返回空符号（`gopls`、`pyright`、`clangd`）或结果有限（`rust-analyzer`）。

这次重构中引入的"动态选择器"将来可能惠及其他选择器，例如允许全局搜索选择器（`<space>/`）在您更改查询时即时更新。

## 版本控制 HEAD 状态栏元素

{{ asciinema(id="vcs-statusline-element", width="94", height="25") }}

添加了一个新的状态栏元素，用于显示版本控制 HEAD。通过在配置的
[`editor.statusline`](https://docs.helix-editor.com/configuration.html#editorstatusline-section)
部分的 `left`、`center` 或 `right` 侧引入 `"version-control"` 元素，将其添加到您的状态栏中。

## 增强键盘协议

现在启用了对[增强键盘协议]的支持。支持该协议的终端（Kitty、WezTerm）现在可以向 Helix 发送明确的按键码，从而启用以前无法绑定的按键组合，如 `C-/`、`C-h` 或 `S-backspace`。对于支持该协议的终端，此功能开箱即用。

[增强键盘协议]: https://sw.kovidgoyal.net/kitty/keyboard-protocol/

## 运行时目录重构

Helix 将 tree-sitter 解析器和查询、主题以及教程文件保存在一个特殊的"运行时"目录中。这个目录通常由包管理器安装，不应手动修改。现在支持多个运行时目录，因此您可以保持包管理器安装的运行时文件不变，同时将新的 tree-sitter 解析器和查询添加到本地运行时目录。在 Unix 系统上，您可以将文件添加到 `~/.config/helix/runtime` 目录，该目录优先于包管理器安装的运行时目录。
