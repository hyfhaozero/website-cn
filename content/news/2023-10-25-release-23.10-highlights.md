Helix 23.10 版本终于来了！23.10 引入了一些非常重大的内部变化，这些变化启用了一些强大的功能。**非常感谢**所有让这个版本成为可能的人。

刚接触 Helix？
Helix 是一个模式文本编辑器，内置支持多重选择、语言服务器协议（LSP）、tree-sitter，以及实验性的调试适配器协议（DAP）支持。

让我们来看看这个版本的亮点。

## 多语言服务器

23.10 对我们处理语言服务器的方式带来了重大改变。以前每种语言一次只能支持一个语言服务器，但现在您可以配置多个语言服务器协同工作。例如，您可以在 JavaScript/TypeScript 开发中使用 TypeScript 语言服务器，同时配置 [efm-langserver](https://github.com/mattn/efm-langserver) 用于 Prettier 格式化和 ESLint 诊断。借助对多语言服务器的新支持，您可以指定从每个语言服务器使用哪些功能。

这对语言配置（`languages.toml`）中语言服务器的配置方式带来了重大改变。所有语言服务器现在都与语言分开指定，然后每种语言选择使用哪个或哪些语言服务器。从上面的例子来看，您可以像这样配置 efm-langserver 和 typescript-language-server：

```toml
[language-server.efm]
command = "efm-langserver"
config.documentFormatting = true

[language-server.typescript-language-server]
command = "typescript-language-server"
args = ["--stdio"]
config.hostInfo = "helix"

[[language]]
name = "typescript"
auto-format = true
language-servers = [
  { name = "efm", only-features = ["format", "diagnostics"] },
  { name = "typescript-language-server", except-features = ["format", "diagnostics"] },
]
```

在[语言配置文档](https://docs.helix-editor.com/languages.html)中查看更多详情。

## 使用 Nucleo 进行模糊匹配

{{ asciinema(id="nucleo-nix-store", width="94", height="25") }}

Helix 使用"模糊"匹配来在文件选择器等组件中过滤输入内容。以前我们使用流行的 `skim`/`fuzzy-matcher` crate，但在 23.10 版本中，我们切换到了新的 [`nucleo`](https://github.com/helix-editor/nucleo) crate。Nucleo 比 skim 和 fzf 快得多，正确处理 Unicode，并且使用了一个感觉更直观的加分系统。

Nucleo 还使我们能够惰性流式传入新项目，这对选择器的用户体验是一个巨大的提升。在上面的 asciicast 中，我正在扫描我电脑的 `/nix/store`，这是一个包含超过两千万个文件的巨大目录。文件选择器现在在我们扫描目录时逐步工作，并在我们找到文件时匹配它们。

Nucleo 也为未来的选择器升级铺平了道路。请关注即将发布的更新说明，看看我们将在哪里改进选择器。

## 智能 Tab

智能 Tab 是默认键映射中绑定到 Tab 键的新功能。当您按下 Tab 并且光标左侧的行不全是空白字符时，光标将跳转到语法树父节点的末尾。例如：

```nix
{
  key = "value";
  nested = {
    key2 = "value2"; # 当光标在这里的行尾时，<tab>
                     # 跳转到下一行右花括号之后。
  };
}
```

这在 Nix 等语言中非常有用，可以在属性集末尾添加分号，或在类 C 语言中跳转到块末尾：

## 对寄存器的扩展支持

{{ asciinema(id="special-registers", width="94", height="25") }}

寄存器允许您保存和粘贴值。例如，您可以选择一个段落，使用 `"ay` 将其拖拽到 `a` 寄存器中，稍后使用 `"ap` 粘贴该段落。然而，某些寄存器在读取或写入时有特殊效果，比如 `_` "黑洞"寄存器：任何写入的值都会被丢弃，也无法读取任何内容。特殊寄存器已经扩展，包括来自 Kakoune 和剪贴板寄存器的一些有用寄存器：

* `%`：当前缓冲区名称
* `#`：每个选择的序号，从 1 开始
* `.`：每个选择的内容
* `*` 和 `+`：系统剪贴板和主剪贴板

另外，请查看新的寄存器状态栏元素。当您用 `"` 选择寄存器时它会显示。注意当我们选择 `#` 寄存器时，状态栏右下角出现 `reg=#`，当我们粘贴该寄存器（`p`）或增加选择（`C-a`）时它消失了。

## 对 LSP DidChangeWatchedFiles 的初始支持

{{ asciinema(id="initial-lsp-didchangewatchedfiles", width="94", height="25") }}

某些语言服务器使用 DidChangeWatchedFiles 通知来发现其他语言中相关文件的变化。例如，当您将新依赖项添加到 `Cargo.toml` 时，`rust-analyzer` 会获取并索引新的依赖项。现在，当文件被 Helix 本身更改时，Helix 将发送文件更改通知。对 DidChangeWatchedFiles 的完整支持需要文件监视器，但目前我们能够支持 DidChangeWatchedFiles 最常见的用例。

## 正则表达式提示的语法高亮

<img src="/regex-prompt-highlighting.png" style="max-width: 80%; display: block; margin-left: auto; margin-right: auto"/>

像使用 `s`、`S` 或 `|` 创建的正则表达式提示现在通过 tree-sitter-regex 进行语法高亮。高亮使特殊字符更加明显，并可以捕获语法错误，如尾随反斜杠。
