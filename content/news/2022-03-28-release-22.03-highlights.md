从提升生活质量的小改进和错误修复，到大型功能特性和重构，Helix 22.03 带来了一些令人兴奋的变化。Helix 是一个模式文本编辑器，内置支持多重选择、语言服务器协议（LSP）、tree-sitter，现在还包括调试适配器协议（DAP）。

在查看亮点之前，有一些管理方面的说明。`master` 分支的更改现在与发布[文档][docs]分开发布。在此处查找新的 `master` 文档[此处][master-docs]。Helix 正在切换版本方案。预计会发现新版本采用日历版本格式：`YY.0M(.MICRO)`。我们计划大约每两个月发布一个常规版本。查看[更新日志][changelog]了解此版本的所有新功能和修复。

说完了这些，让我们来看看亮点吧！

## 健康检查

Helix 22.03 带来了一个新的 CLI 标志：`hx --health`。使用新的健康检查标志来排查缺失的语言服务器和查询问题。

使用 `hx --health` 检查所有语言的健康状况，或使用 `hx --health <lang>` 查询特定语言的详细信息。

## 实验性 DAP 支持

调试适配器协议（DAP）是编辑器和调试器之间通信的抽象协议。它在精神上与语言服务器协议（LSP）非常相似，但专为调试而构建。使用 `<space-d>` 与调试适配器交互。

{{ asciinema(id="dap", width="154", height="46") }}

请注意，DAP 适配器是实验性的：它尚未有文档记录，可能存在错误，并且用户体验有点笨拙。非常欢迎贡献代码！

## 增量注入解析重写

由 tree-sitter 驱动的语法高亮的一个比较酷的功能是能够将一种语言*注入*到另一种语言的文档中。例如，如果您正在编写 Markdown，您可能会使用如下代码围栏：

    这是一些 Rust 代码：

    ```rust
    println!("Hello, world!")
    ```

Helix 通过注入 tree-sitter-rust 来高亮 Rust 代码块。注入功能已被重写，以便其中的更改现在可以增量解析，这在编辑包含大型注入块的文档时大大提高了速度。

伴随这次重写，Helix 现在支持*组合*注入。例如，在突出显示 Interactive Elixir (IEx) 时，我们可能有如下代码块：

```elixir
iex> send(self(), :hello)
iex> receive do: (:hello -> :ok)
```

IEx 语法在提示符标记后将 tree-sitter-elixir 注入到每一行。组合注入覆盖了必须在一个组合文档中解析多个独立注入文档的情况，例如：

```elixir
iex> if true do
...>   :ok
...> end
```

Helix 现在可以一起解析所有三行。将来，这可用于增加对 EJS 或 ERB 等模板语言的支持。

## Tree-Sitter 语法重构

过去，tree-sitter 语法仓库作为 Git 子模块添加到 Helix 仓库中。然而，子模块可能难以使用，尤其是在仓库中有超过 50 个子模块的情况下。随着语言支持的改进，克隆、CI 和打包时间变慢了。

Helix 22.03 完全改造了 tree-sitter 语法的系统。最引人注目的是**子模块不见了**！如果您正在处理源代码，您现在可以使用标准的 `git clone https://github.com/helix-editor/helix` 进行克隆。那么 tree-sitter 子模块去哪了？它们已被两个新的 CLI 标志所取代：`hx --grammar fetch` 用于将语法仓库克隆到运行时目录，以及 `hx --grammar build` 用于编译它们。使用 `-g` 作为缩写。语法仓库以并行方式浅层克隆，因此在网络连接良好的情况下，获取所有 60 个语法现在只需 6 秒钟。

如果您从源代码构建或开发 Helix，请注意获取和构建已包含在 `helix-term` 构建步骤中，因此您通常不需要手动获取或构建语法。

此外，如果您想自定义获取和构建哪些 tree-sitter 语法，您现在可以在您的 `languages.toml` **顶部**添加 `use-grammars` 键：

{{ asciinema(id="use-grammars", width="92", height="24") }}

如果您正在编写 tree-sitter 语法，您可以通过在 `languages.toml` 中将 Helix 指向您语法的本地路径来尝试集成它，而无需将更改发布到 Git 远程：

```toml
[[language]]
name = "mylang"
# ..

[[grammar]]
name = "mylang"
source = { path = "/local/path/to/tree-sitter-mylang" }
```

运行 `hx -g build` 将构建该语法。添加一些查询，您就可以开始交互式语法开发了。
