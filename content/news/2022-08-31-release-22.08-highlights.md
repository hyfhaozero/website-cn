Helix 是一个模式文本编辑器，内置支持多重选择、语言服务器协议（LSP）、tree-sitter，以及实验性的调试适配器协议（DAP）支持。

今天我们迎来了 22.08 版本的发布，这个版本包含了大量的修复和新功能。**非常感谢**我们的贡献者！这个版本有 87 位贡献者参与。🎉

让我们来看看 22.08 的亮点功能。

## 缩进引导线

{{ asciinema(id="indent-guides", width="94", height="25") }}

缩进引导线提供了当前缩进级别的可视化表示。通过 `editor.indent-guides.render` 键启用缩进引导线。用作引导线的字符也可以自定义。

## 光标行

{{ asciinema(id="cursorline", width="94", height="25") }}

光标行是跟随光标移动的水平高亮条。光标行可以为主光标和所有次光标设置主题，并且可以通过 `editor.cursorline` 选项独立于主题进行启用或禁用。

## 模式颜色

{{ asciinema(id="color-modes", width="94", height="25") }}

状态栏中的模式指示器现在可以根据当前模式进行样式设置。此功能可通过 `editor.color-modes` 选项启用，颜色可以在主题中使用 `ui.statusline.{insert,normal,select}` 键进行配置。

## 可配置的状态栏

{{ asciinema(id="configurable-statusline", width="94", height="25") }}

状态栏现在可以在配置的 `editor.statusline` 部分中进行配置。元素可以放置在状态栏的左侧、居中或右侧，并带有可配置的分隔符和间距。现在还可以在状态栏中添加两个新元素：`file-line-ending` 和 `position-percentage`。

## LSP 签名帮助

<img src="/signature-help.gif" style="max-width: 80%; display: block; margin-left: auto; margin-right: auto"/>

签名帮助在您输入函数调用的参数时提供文档说明，并跟踪当前正在输入的参数是哪个。签名帮助默认启用。

## LSP 文档高亮

{{ asciinema(id="document-highlight", width="94", height="25") }}

文档高亮请求（`Space-h`）会为主光标下的所有符号实例创建选择。Helix 对多重选择有强大的支持，因此您可以同时编辑所有选择（例如使用 `c`）或在选择之间循环（`(`/`)`）。

## LSP 诊断信息选择器

{{ asciinema(id="diagnostic-picker", width="94", height="25") }}

新的缓冲区和工作区诊断信息选择器可用于跳转到语言服务器的诊断信息，如警告和错误。使用 `Space-g` 打开当前缓冲区的诊断信息选择器，使用 `Space-G` 查看工作区中的所有诊断信息。

## 跳转列表选择器

{{ asciinema(id="jumplist-picker", width="94", height="25") }}

跳转列表保存了选择的历史记录。使用 `C-s` 保存选择，使用 `C-i` 向前跳转，使用 `C-o` 向后跳转。跳转列表是一个强大的工具，特别是在使用语言服务器的跳转到定义或全局搜索时，它们都会自动保存到跳转列表中。

22.08 添加了一个新的选择器，可用于在跳转列表的条目之间跳转。预览窗格显示已保存的主选择所在的行。使用 `Space-j` 打开跳转列表选择器。

## 外部格式化工具

{{ asciinema(id="external-formatter", width="94", height="25") }}

许多语言服务器提供保存时格式化的功能。对于没有语言服务器的语言，或者对于未实现格式化的语言服务器，现在可以配置外部格式化工具二进制文件。文档会通过格式化工具的标准输入传递，并替换为标准输出的格式化结果。

当您更喜欢外部工具的格式化效果而非语言服务器提供的格式化时，也可以使用外部格式化工具。例如，如果您为 Python 格式化配置了 `black`，格式化将通过 `black` 完成，而不是 `pylsp`。

## 括号粘贴

{{ asciinema(id="bracketed-paste", width="94", height="25") }}

括号粘贴是终端模拟器的一项功能，允许终端程序识别粘贴序列并处理粘贴的文本。如果没有括号粘贴支持，通过操作系统级别的粘贴（`C-v`）粘贴的文本在 Helix 看来就像是快速输入的文本，这会导致一些奇怪的副作用，比如尴尬的缩进或神秘出现的自动配对字符。

22.08 增加了对括号粘贴的支持，所以现在从支持括号粘贴的终端模拟器粘贴到 Helix 的所有内容，都像是在普通模式下按了 `Space-p` 一样。在上面的演示中，整本《白鲸记》的文本在插入模式下通过 `C-v` 瞬间粘贴完成。
